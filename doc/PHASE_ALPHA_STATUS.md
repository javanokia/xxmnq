## 技术决策记录

### 为何采用部分修改+多语言文件方案？

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| 直接修改JS | 快速 | 不可维护，难以切换语言 | ✗ |
| 修改语言PO文件 | 符合ADR架构，易维护 | 较慢 | ✓ 推荐 |
| 混合方案（现在） | 快速验证 | 混乱，中间形态 | ⚠ 过渡状态 |

### 后续建议

**Phase 1.1** 使用多语言工具链完成所有文本国际化：
```bash
xgettext script/*.js -o lang/adarkroom.pot
cd lang && for d in zh_cn/; do 
  msgmerge -U "$d/strings.po" adarkroom.pot
done
```

这样可以：
- 维持代码原始完整性
- 支持一键切换中文/英文
- 便于未来扩展其他语言
