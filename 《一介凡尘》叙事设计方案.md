# 《一介凡尘》叙事设计方案

> 叙事设计师交付版 · v1.0
> 项目：xxmnq（一介凡尘）
> 基础引擎：A Dark Room (doublespeakgames)
> 日期：2026-04-01

---

## 一、设计愿景

**一句话：** 在一个灵气枯竭的世界里，守住一堆火，等待一个不可能的渡劫。

这不是一个关于修仙的故事。这是一个关于**在绝望中继续燃烧**的故事。

修仙是外壳——获得力量、逃避死亡、飞升虚空。
末法是隐喻——失去秩序之后，人如何重建意义。

玩家不只是管理者。玩家是一个**正在思考"为什么还要活下去"的人**，而火、营地、人群、天劫——都是这个内心问题的投影。

---

## 二、核心叙事主题

### 主题 1：末法时代的幸存者哲学

末法之前，有人飞天遁地，有宗门千万。现在全没了。

这不是"悲剧"，而是——**历史的常态**。文明会兴衰，灵气会枯竭，任何强大都有终点。

玩家面对的不是一个需要"解决"的问题，而是一个需要"接受"的现实。

**叙事策略：** 不要用文字直接说明这一点。让玩家在事件中自己感受到——废墟比活人更有故事感，帮助过你的人没有回来，承诺渡劫成功就能改变一切，但其实什么都没改变。

### 主题 2：火的隐喻

火 = 生存 / 灵气 / 希望 / 记忆 / 传承

从"续添灵晶"到"渡劫升空"，火始终在场。它是该游戏最强大的叙事工具，因为它从始至终都和玩家在一起。

**叙事策略：** 在关键时刻，让火的状态呼应玩家的内心状态。

```
游戏初期：        "炉火摇曳不定。风吹进来的时候，你紧了紧衣襟。"
营地建立后：      "炉火烧得正旺。你开始觉得，这里也许能待一阵子。"
渡劫前夜：        "火很安静。比往常更安静。你添了一根灵晶。"
渡劫成功：        "炉火熄灭。但你不再需要它了。"
```

### 主题 3：轮回不只是机制

原版 ADR 的 Prestige = 重复游玩加成。
《一介凡尘》的 Prestige = **"我是不是在重复同一天？"**

当玩家第二轮、第三轮回到同一个山洞，他们的"既视感"会越来越强。最终，他们开始质疑：**这个循环本身，是不是问题的一部分？**

**叙事策略：** 增加周目间的叙事呼应。

```
第一轮："这是第一次见到她。"
第三轮："这是第三次见到她。你开始知道她下一句要说什么。"
第五轮："第五次。你突然意识到——你比她活得更久，这是不是一种诅咒？"
```

## 三、叙事结构分层

### Layer 0：环境叙事（无玩家选择）

这是最底层，也是渗透感最强的层。

**实现方式：** 在各个模块的文本描述中，持续注入末法世界观元素。

```javascript
// room.js 改造示例
"the room is dark": "洞穴漆黑，只有炉火在燃烧。",
"the room is freezing": "寒气从石缝渗入。末法之后，天不再照顾人。",
"the room is warm": "炉火将寒气挡在石壁之外。仅此而已。",

// outside.js 改造示例
"A Silent Forest": "灵林沉寂。末法前，这里有鸟鸣、兽吼、修士的飞剑声。现在只剩风声。",
"village is growing": "营地又多了几座茅庐。他们管这里叫'家'——在末法时代，这个词比任何丹药都珍贵。",
```

### Layer 1：随机事件叙事（一次性选择）

当前 cultivation.js 中的事件属于这一层。它们有叙事，但没有积累。

**改进方向：** 增加结果的延迟性和关联性。

```
事件触发 → 选择 → 
  ├→ 即时结果（立即反馈）
  ├→ 延迟结果（3-5天后才显现）
  └→ 叙事记忆（影响未来事件的触发条件和文本）
```

### Layer 2：NPC 关系叙事（跨事件积累）

这是当前最薄弱的层。

**改进方向：** 为每个主要 NPC 建立独立的关系线和叙事档案。

```javascript
// NPC 叙事档案结构示例
NPCProfile = {
    name: "老散修",
    story: [
        {
            trigger: "game.builder.level >= 1",
            text: "他只是缩在角落发抖。"
        },
        {
            trigger: "game.builder.level >= 3",
            text: "他开始帮忙整理洞穴。",
            action: "解锁建造功能"
        },
        {
            trigger: "event.talked_about_past === true",
            text: "他有时会提起——很久以前，有人和他一起烧过火。"
        },
        {
            trigger: "game.builder.level >= 5",
            text: "他看着你，目光里有某种东西。不是希望，是——承认。"
        },
        {
            trigger: "tribulation.imminent",
            text: "他没有送行。只是点了点头。你知道这就够了。"
        }
    ]
}
```

### Layer 3：主角内心叙事（无声的旁白）

这是整个叙事架构的顶层，也是玩家最感知不到、但影响最深的层。

**核心思路：** 玩家从来不直接控制主角说什么。但系统在关键时刻，插入主角的内心独白。

```javascript
// 示例：游戏时间的每个"阶段节点"插入内心独白
gamePhaseNarrative = {
    // 第一个月结束时
    "month1_complete": [
        "你活过了一个月。",
        "不是所有人都做到了。",
        "但你开始觉得，这不算什么成就——这只是，继续活着。",
        "也许这就够了。也许不够。"
    ],
    
    // 人口超过10人
    "population_10": [
        "十个人。",
        "他们都有自己的名字，但你不一定记得住。",
        "他们在等你做决定。",
        "有时候你会想：我凭什么？"
    ],
    
    // 第一次有人死去
    "first_death": [
        "营地又空了张床。",
        "没有人哭。不是因为不在乎，是因为——没时间哭。",
        "火还要继续烧。"
    ],
    
    // 渡劫前夜（特殊）
    "tribulation_eve": [
        "明天就要出发了。",
        "你看着营地。茅庐、火、睡着的人。",
        "你想起第一天来的时候，这里只有一个发抖的老散修。",
        "现在他们有十个人。一座营地。一些灵晶。",
        "你不知道这算不算'成功'。",
        "但你知道——火还能烧下去。",
        "天亮了。"
    ]
}
```

## 四、核心事件方案

### 4.1 新增：主角内心叙事事件

#### 事件1：深夜独坐（触发：第5天起，每隔7-10天）

```javascript
{
    title: "深夜",
    isAvailable: function() {
        return Engine.activeModule == Room &&
            $SM.get('game.day', true) >= 5 &&
            $SM.get('game.temperature.value', true) >= 2;
    },
    scenes: {
        'start': {
            text: [
                '营地都睡了。',
                '只有你坐在火边。',
                '在想什么，或者什么都没想。'
            ],
            notification: '夜深了',
            blink: false,  // 不显眼，只在玩家打开界面时出现
            buttons: {
                'think': {
                    text: '想一些事情',
                    nextScene: 'reflection'
                },
                'sleep': {
                    text: '回去睡',
                    nextScene: 'end'
                }
            }
        },
        'reflection': {
            text: function() {
                var day = $SM.get('game.day', true);
                var population = $SM.get('game.population', true);
                
                if (population >= 5) {
                    return [
                        '他们都在这里。',
                        '老人、年轻人、孩子。',
                        '他们在等你做决定。',
                        '你从来没有被这么多人需要过。',
                        '也没有这么害怕过。'
                    ];
                } else if (day >= 30) {
                    return [
                        '已经' + day + '天了。',
                        '你还记得来时的路吗？',
                        '不记得了。',
                        '也许不重要。'
                    ];
                } else {
                    return [
                        '火在烧。',
                        '夜很长。',
                        '你把一根灵晶添进火里。'
                    ];
                }
            },
            buttons: { 'leave': { text: '睡去', nextScene: 'end' } }
        }
    }
}
```

## 六、结局文案体系

### 结局 A：渡劫成功

```
你不知道自己是死了，还是活了。

没有身体的感觉。没有时间的流逝。
只有一种奇异的——空。

不是虚空的空。是填满了之后，
反而不知道该装什么的空。

你睁开眼睛。
这是——一个新的世界。
有光。有树。有流动的东西。
不是灵气。但也是某种能量。

你在地上坐了一会儿。
然后站起来。

该做什么呢？

你不知道。
但这次，你有足够的时间去想。
```

### 结局 B：渡劫失败

```
天舟碎了。
不是戏剧性的爆炸。
是慢慢地，一点一点地，瓦解。

你想起了营地。
想起了老散修最后的点头。
想起了添灵晶的声音。
想起了洞口的那些面孔。

他们会继续活下去。
就像你继续活过的那五十年一样。

这就是尽头。

不坏。
也不够好。
只是——尽头。

你闭上眼睛。
火灭了。
```

### 结局 C：轮回（Prestige）

```
意识在消散。
但不是——彻底的消散。

你看见了。
另一座山洞。
另一堆火。
另一个坐在火边的人。

那个人抬起头。
隔着不知道多少轮回的距离，
你看见了自己的脸。

你知道他会问什么问题。
你知道他会得到什么回答。
你知道故事会怎么结束。

但你什么都不说。
什么都不做。

只是看着。

然后——

洞穴还是这个洞穴。
火还是快要熄灭的火。
但这次，你知道火是怎么烧的了。

你知道明天会发生什么。
你知道第十五天会有一个求道者来问修炼之法。
你知道第三十天天空会有一道细缝。

你知道。

然后你添了一根灵晶。

火继续烧。
```

## 八、技术实现注意事项

### 事件注册

所有新增事件应追加到对应的模块事件列表：

```javascript
// 追加到 cultivation_extended.js 或新建 narrative_main.js
if (typeof Events !== 'undefined' && Events.Room) {
    Events.Room = Events.Room.concat(narrativeEvents);
}

// 野外事件追加到 world_narrative.js
if (typeof Events !== 'undefined' && Events.Outside) {
    Events.Outside = Events.Outside.concat(outsideNarrativeEvents);
}
```

### 事件标记系统

建议统一使用 `$SM` 系统存储事件状态：

```javascript
// 标记事件已触发
$SM.set('event.{eventId}_shown', true);

// 存储选择结果（用于联动）
$SM.set('event.{choiceMade}', true);

// 存储延迟触发的标记
Events.saveDelay(function() {
    $SM.set('event.delayed_' + eventId, true);
}, 'delayed_trigger_key', delaySeconds * 1000);
```

### 文本动态生成

使用函数作为 `text` 字段，可以根据游戏状态生成不同文案：

```javascript
text: function() {
    var population = $SM.get('game.population', true);
    if (population >= 10) {
        return ['十个人了。', '他们都在看着你。'];
    } else {
        return ['火还在烧。'];
    }
}
```

## 九、总结

《一介凡尘》的叙事潜力远未充分释放。

当前版本是一个**功能完整的修仙改造**。
下一版本的目标是成为**一个让人记住的叙事体验**。

核心原则只有三条：

1. **火比语言更有说服力**——让火的视觉/数值状态反映叙事状态
2. **选择要有重量**——每个选择都应该有延迟后果
3. **不说教，只呈现**——末法时代的残酷和希望，都让玩家自己感受
